Application Layer

Layer 7 | Application Layer

Sending (Encapsulation) [Step One]

This is the topmost layer where you, the user, interact with an app like YouTube, Gmail, or a game.

  • Not the app itself but the protocols behind it (like HTTP for websites, SMTP for mail, or FTP for file sharing).
  • Prepares what kind of data needs to be sent: a file, message, or request.
  • Uses protocols to request what format and service is needed:
    • HTTP to view a webpage
    • VoIP to send audio/video data
    • DNS to look up domain names
  • Sends the data to Layer 6 for formatting, encryption, or compression.

Example: Clicking a link on a website triggers an HTTP request at this layer.

Receiving (Decapsulation)

The final layer where the user sees or interacts with the result.

  • Receives the cleaned, structured, and translated data from Layer 6.
  • Delivers it to the app like showing the webpage or loading the image.
  • Lets the app decide what to do next: open it, show it, or respond to it.
  • Completes the full circle of communication, from the sender’s click to the receiver’s screen.

Example: You receive a video stream in YouTube, or your Gmail inbox refreshes with new messages.
