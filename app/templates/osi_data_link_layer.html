Data Link Layer (MAC Address Layer)

Encapsulation

  • This layer adds the MAC address a name tag used only inside the local network (LAN).
  • MAC addresses are physical, unique hardware IDs built into each devices network card AKA NIC (like AA:BB:CC:DD:EE:FF).

  • The Data Link Layer mixes with your packer (from layer 3) and puts it inside a frame and gives it:
    • Source MAC: Your device’s MAC address
    • Destination MAC: The Destinations MAC (If its not in the LAN, it sends it to the router)

  • This layer is in charge of local delivery, it only cares about getting the data from one device to the destination (like to a router or destination device on the LAN).
  • If the destination is outside your network, the MAC address is set to the router’s MAC so it can forward it further.

Examples

  • Source MAC: 00:1A:2B:3C:4D:5E
  • Destination MAC: 00:1F:22:33:44:55 (router)
  • Encapsulation: IP packet → Frame (with MAC header)
  • Devices: Switches, Network Interface Cards (NICs)
