Session Layer

Layer 5 | Session Layer

Sending (Encapsulation)

This layer manages the full session between two devices, it decides:

  • When the session starts and ends (like logging in and logging out).
  • Whether the communication is two-way at the same time (full duplex) or takes turns (half duplex).
  • When to pause or resume a connection especially during interruptions.
  • Hands off a clean session for Layer 6 to format and send data.

Example: Logging in to the bank ,the Session Layer starts a secure session that will stay open while you infact "bank"

Receiving (Decapsulation)

This layer is responsible for keeping the session in sync on the receiving end. It:

  • Sees a session exists, and allows communication to continue smoothly.
  • Handles unexpected pauses and resumes data when possible.
  • Decides when the session should end (like after logging out or inactivity).
  • Pushes it up to the Presentation Layer.

Example: The bank server accepts your login session and keeps it alive while you navigate between pages.
