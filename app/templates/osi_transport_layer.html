Sending (Encapsulation)

This layer breaks the message into smaller segments and adds info (akin to mailing info) to help the destination put it all back together. It also manages reliability and uses ports to direct data to the correct application (like a browser or video game).

  • Adds a TCP or UDP header, depending on which type of connection is needed:
    • TCP (Transmission Control Protocol): Slower but makes sure the packet is correct, or is resent
    • UDP (User Datagram Protocol): Fast but does not check if the packet is correct, never resent.
  • Adds port numbers:
    • Source port: The app on your device sending the data
    • Destination port: The app/service you’re talking to (like port 80 for websites, 443 for secure web, or 53 for DNS)
  • Passes the segment to Layer 3 (Network Layer) for delivery across networks.

Receiving (Decapsulation)

The receiving device uses this layer to figure out which application should get the data and whether it arrived correctly.

  • Checks the destination port to figure out which application to send the data to (e.g. browser, game, etc.).
  • Removes the TCP or UDP header.
  • Passes the reassembled message (TCP reassembles, UDP is simply hoping it's ordered) up to Layer 5 (Session Layer).
