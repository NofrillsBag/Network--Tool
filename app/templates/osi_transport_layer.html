Transport Layer

Encapsulation

  • The transport layers job is to break data into chunks and keep track of those pieces during delivery.
  • Each seperate piece is given something called a transport header this holds info like what order the pieces go in and how to reassemble them.
  • You can think of headers as writing labels on each box in a delivery truck.

  • This layer also decides which way it wants to piece up the data, this is the protocol to use:

    • TCP (Transmission Control Protocol): Reliable and safe. Checks if the data arrived, resends if it didn’t. Slower, but guaranteed.
    • UDP (User Datagram Protocol): Fast and lightweight. Doesn’t check if the data made it — good for live voice or video.


  • So if youre gaming or on a video call, UDP might be used it keeps going no matter what. If you’re sending an email or downloading a file, TCP ensures nothing is lost.

Examples

  • TCP: Emails, File Downloads, Web Pages
  • UDP: Online Games, VoIP, Livestreams
  • Headers: Sequence Numbers, Port Numbers (where to enter from)
